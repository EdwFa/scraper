# Выгрузка уведомлений ЕРВК (Парсер)

Этот проект представляет собой веб-приложение на базе **Streamlit**, предназначенное для автоматизированного скоростного сбора (парсинга) расширенных данных уведомлений с публичного портала Единого реестра видов контроля (ЕРВК) — `https://ervk.gov.ru/public/notices`.

## Особенности (API Версия)
Сайт ЕРВК защищает свои данные, используя динамические одноразовые токены, которые быстро протухают. Ранее скрипт "прокликивал" страницы в браузере, но теперь используется **гибридная сверхбыстрая архитектура**:
1. **Playwright** открывает сайт в скрытом режиме (headless) и эмулирует действия пользователя для настройки фильтров ("Вид деятельности", "Субъект РФ").
2. В фоне перехватывается секретный `Token` авторизации.
3. Осуществляется прямое пакетное скачивание чистого JSON через высокопроизводительную библиотеку **aiohttp** без медленного рендеринга HTML.
4. Выполняются глубокие запросы к скрытому API портала (`/portal/public/notices/{id}`) для извлечения расширенной информации, которой нет в общем списке (ИНН, ОГРН, формы собственности).
5. Встроен механизм автообновления: если во время длительной работы токен протухает (ошибка 401), фоновый браузер автоматически запрашивает новый токен, и скрипт продолжает скачивание без обрывов.

## Стек технологий
* **Python 3**
* **Streamlit** — создание графического веб-интерфейса.
* **Playwright (async)** — автоматизация Chromium-браузера и перехват токенов авторизации.
* **aiohttp** — сверхбыстрые асинхронные запросы напрямую к API сайта.

## Структура собираемых данных
Для каждой карточки программа формирует JSON со следующими полями:
* `номер уведомления`
* `наименование хозяйствующего субъекта`
* `дата`
* `наименование работы или услуги`
* `коды ОКВЭД`
* `Субъект РФ`
* `Адрес места осуществления деятельности`
* `Наименование контрольного органа`
* `ИНН` (выведен в корень для удобства)
* `Тип` (Юридические лица / Индивидуальные предприниматели)
* `Контролируемое лицо` — вложенный объект с детализированной информацией (в зависимости от типа лица):
  * **Для ЮЛ**: Полное наименование, Краткое наименование, Адрес, ИНН, ОГРН.
  * **Для ИП**: ФИО, ИНН, ОГРНИП.

## Установка и настройка

1. Убедитесь, что у вас установлен Python (рекомендуется версия 3.8+).
2. Склонируйте репозиторий к себе на компьютер:
   ```bash
   git clone https://github.com/EdwFa/scraper.git
   ```
3. Перейдите в папку с проектом:
   ```bash
   cd scraper
   ```
4. Создайте и активируйте виртуальное окружение:
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   ```
5. Установите зависимости:
   ```bash
   pip install streamlit playwright aiohttp
   ```
6. Установите необходимые бинарные файлы браузеров для Playwright:
   ```bash
   playwright install chromium
   ```

## Запуск приложения

Для запуска интерфейса выполните команду:
```bash
streamlit run app.py
```
После этого в вашем браузере по умолчанию автоматически откроется вкладка с приложением (обычно `http://localhost:8501`).

## Использование
1. При необходимости скорректируйте параметры поиска в полях ввода на веб-странице.
2. Нажмите синюю кнопку **«Начать выгрузку»**.
3. В интерфейсе будет отображаться статус загрузки, количество найденных совпадений и прогресс скачивания батчей.
4. Процесс полностью автоматизирован: если токен устареет, скрипт обновит его самостоятельно.
5. По завершении обхода программа выведет первые несколько записей для предпросмотра.
6. Нажмите кнопку **«Скачать notices.json»**, чтобы сохранить все собранные данные к себе на компьютер, либо найдите файл `notices.json` в корневой директории проекта.

## Логирование
Процесс работы программы автоматически фиксируется в файле `scraper.log` (создается в папке проекта при первом запуске). 
В логах сохраняется следующая информация:
* Дата и время запуска и окончания работы парсера.
* Введенные параметры фильтров поиска.
* Процесс получения, "протухания" (ошибка 401) и успешного автообновления токена авторизации.
* Любые сетевые ошибки и статистика сбора данных.
